## Concepts that I have understand today:

### 🔐 Cross Account Access

Cross Account Access allows one AWS account to securely access resources in another AWS account using IAM Roles and Trust Policies.

👉 Key Point:
Access is temporary and role-based, not based on shared passwords.

---

### 🪣 Amazon S3 (Simple Storage Service)

Amazon S3 is a cloud storage service used to store and manage files like images, backups, and documents.

👉 Learned:

* Creating S3 buckets
* Setting permissions
* Basic access control using policies

👉 Key Point:
S3 = scalable and secure cloud storage

## 💡 Real-World Scenario

In real AWS environments:

* One account stores data in S3
* Another account accesses it securely using IAM Roles
* No credentials are shared manually

This is how secure cloud systems are built.
